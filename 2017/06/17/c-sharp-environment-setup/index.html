C# / .NET Environment Setup

Building C# / .NET Applications with Visual Studio Code

Brief overview: .NET is a framework included with Visual Studio Code (VS Code) that provides a variety of services/interfaces. C# is a language that applications can be written in. VS Code runs C# applications on top of the .NET framework.

Download Visual Studio Code and the C# Extension

Navigate here to download and install VS Code.

Once VS Code has been installed on your machine, open it up and click the square-like icon on the left-hand side of the window. Search for ‘C#’ in the search bar that pops out, then click to install the extension (it should be the first on the list). You may need to restart VS Code to enable it.

Install .NET Core

Follow the instructions here to install .NET Core.

NOTE: If you’re running macOS Sierra, you may encounter an error at Step 3 of their instructions. If you get a message in Terminal that says “dotnet: command not found”, run the command ln -s /usr/local/share/dotnet/dotnet /usr/local/bin.

You should then be able to run through Step 3 of their instructions to produce the “Hello World!” output in your Terminal window.

Try Running “Hello, World!” in VS Code

Open up VS Code and the ‘hwapp’ project folder that was just generated in the previous step - this can either be done from the VS Code menu or by running the command code . in Terminal after navigating into the ‘hwapp’ directory.

Once the project folder is open in VS Code, select ‘Debug > Start Without Debugging’ from navigation bar at the top of your screen and select ‘.NET Core’ from the dropdown that appears. This will generate a .vscode/ folder within your project and a launch.json file within that folder.

Open that launch.json file and change the value of the “program” key. By default it reads:

<target-framework>/<project-name.dll> is a placeholder, and needs to be replaced by the path to the project’s dll file. In this case it should be replaced by netcoreapp1.1/hwapp.dll, which can be found in the /bin/Debug folder of your project directory.

Once this is complete, select ‘Debug > Start Without Debugging’ from navigation bar at the top of your screen again. An error may pop-up that says “Could not find preLaunchTask ‘build’.” If this occurs, select ‘Configure Task Runner’ from the options provided and select ‘.NET Core’ from the dropdown that appears. This generates a tasks.json file.

After the tasks.json file has been generated, you can then select ‘Debug > Start Without Debugging’ from navigation bar at the top of your screen one last time, which will build/run the application. Click the icons in the bottom left of your VS Code window (an ‘x’ within a circle and the caution symbol) and open the ‘DEBUG CONSOLE’ to see your application’s output. Assuming you see the message “Hello World!”, you now have VS Code properly setup to build C# / .NET applications. Congratulations!
