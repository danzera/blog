ESLint Setup

Incorporating ESLint into an AngularJS / Node.js Application

ESLint is an open source, pluggable linting utility for JavaScript. It’s a great way to ensure all members of a project team use a consistent linter and makes it possible to enforce consistent code standards across a project. This post will walk through the ESLint setup process for an AngularJS / Node.js project.

IDE Setup

Download the appropriate ESLint package for your IDE. The recommended packages for most common code editors are listed here.

Install ESLint

Assuming that you already have Node installed and a package.json has been created for your project, run npm install --save-dev eslint eslint-plugin-angular to install ESLint locally. If desired, it can also be installed globally.

Create a .eslintrc File

In the root directory of your project, create a file named .eslintrc which is composed of a configuration object. Here is some sample code for the file:

{
  "extends": ["plugin:angular/johnpapa", "eslint:recommended"],
  "env": { // indicates environment(s) where the code will be used
    "browser": true,
    "node": true
  },
  "globals": { // global variables your script accesses during execution
    "angular": true
  },
  "rules": { // extend (or override) specific rules
    "no-unused-vars": "warn",
    "quotes": [2, "single"],
    "semi": ["warn", "always"],
    "no-console": ["warn", {"allow": ["log", "warn", "error"]}],
    "no-trailing-spaces": "error"
  }
}
  • “extends” can either be a string that specificies a configuration or an array of strings where each additional configuration extends the preceding configurations.
    • “plugin:angular/johnpapa” incorporates some rules from John Papa’s AngularJS style guide. However, the AngularJS Plugin can also be used without the shareable configuration.
      • No rules are enabled ‘out of the box’ with ESLint - “eslint:recommended” enables a subset of ESLint’s core rules that report common problems in JavaScript applications. Rules with check marks next to them in the rules section of the ESLint Documentation indicate those that are enabled by this extension.
  • “env” indicates any environments in which the code will be run, which causes ESLint to allow a predetermined set of global variables specific to those environments.
  • “globals” speficies any additional global variables that your scripts access during execution
  • “rules” allows you to enable additional rules and extend, or override, the set of rules already specified from base configurations. The enforcement severity of rules can also be changed without changing their options. Additionally, you can write your own rules if you’re so inclined. See the ESLint documentation for the full skinny on working with rules.
